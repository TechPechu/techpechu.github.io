Nginx - Week3

Host secure Webpage, Redirect & Rewrite

Prerequisite :

Create a Self signed certificate / Procure a certificate from valid CA

Steps to Create Self-signed certificate

[root] $ openssl req -x509 -nodes -days 365 \
  -newkey rsa:2048 \
  -keyout /etc/nginx/ssl/private.key \
  -out /etc/nginx/ssl/public.pem
Place the certificate in /etc/nginx/ssl

Enabling SSL on Nginx

Edit the domain conf file

vim /etc/nginx/conf.d/techpechu.com.conf
server {
    server_name techpechu.com www.techpechu.com;
    listen 80;
    listen 443 ssl;
    ssl_certificate /etc/nginx/ssl/public.pem;
    ssl_certificate_key /etc/nginx/ssl/private.key;
    root /var/www/techpechu.com/html/;
}

Redirect at Nginx

What is Redirect? Takes user from one URL to other Types of Redirect 301 and 302 are most commonly used. 301 → Permanent Redirect 302 → Temporary Redirect

Syntax for Redirect

reuturn <<statuscode>> <<destination url>>;

To Redirect /abc to /

Edit the domain conf file

vim /etc/nginx/conf.d/techpechu.com.conf
server {
    server_name techpechu.com www.techpechu.com;
    listen 443 ssl;
    ssl_certificate /etc/nginx/ssl/public.pem;
    ssl_certificate_key /etc/nginx/ssl/private.key;
    root /var/www/techpechu.com/html/;
    location =/abc{
    return 301 https://$host/;
    }
}

Rewrite

Actual back path will be masked . Usecase: → To tidy up URL for user experience → Server content from alternate folder on backend

Syntax for rewrite

rewrite <<regex>> [flag];

Flag can be :
last 
break
redirect

Edit the domain conf file

vim /etc/nginx/conf.d/techpechu.com.conf
server {
    server_name techpechu.com www.techpechu.com;
    listen 443 ssl;
    ssl_certificate /etc/nginx/ssl/public.pem;
    ssl_certificate_key /etc/nginx/ssl/private.key;
    root /var/www/techpechu.com/html/;
    location =/abc{
    return 301 https://$host/;
    }
    rewrite ^(/techpechu) / last;
}
server {
     server_name techpechu.com www.techpechu.com;
     listen 80;
     return 301 https://$host$request_uri;

}
